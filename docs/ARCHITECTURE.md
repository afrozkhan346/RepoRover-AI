## ğŸ¯ Design Decisions

### **Why Microservices?**

1. **Independent Scaling** - Each service scales based on its own load
2. **Fault Isolation** - Failure in one service doesn't affect others
3. **Technology Flexibility** - Different services can use different tools
4. **Team Autonomy** - Teams can work independently on services
5. **Cost Optimization** - Pay only for what each service uses

### **Why Cloud Run?**

1. **Zero Infrastructure Management** - No servers to maintain
2. **Auto-Scaling** - Handles traffic spikes automatically
3. **Cost Efficient** - Scales to zero when idle
4. **Fast Deployment** - Minutes to deploy changes
5. **Built-in Features** - HTTPS, logging, monitoring included

### **Why Firestore + Turso?**

1. **Firestore** - Real-time sync, NoSQL flexibility, serverless
2. **Turso** - SQLite compatibility, complex queries, edge replication
3. **Best of Both** - Use right database for each use case

### **Why Redis Cache?**

1. **Performance** - 15-22x faster for repeated queries
2. **Cost Reduction** - Fewer database reads
3. **User Experience** - Instant response times
4. **API Rate Limits** - Reduce external API calls
